fstream

The ability to write and read from files is provided using the fstream library in C++. Similar to using std::ostream& operator<<(...) to overload the stream output methods, the fstream methods allow the user to output the string to a file. Instead of outputting to the terminal (stdout), the user can choose to output a line (terminated by either a newline (\n) or the end of the file) to a file instead. This can be achieved using std::fstream::open:

std::fstream::open

#include <fstream>

std::fstream important_data("/path/to/my/file")

By default, opening the file as above will not create the file or add the contents as required, however. This is due to not setting appropriate permissions for the file stream object which is constructed. A second argument, which outlines the permissions of the file stream, is required. If the file is intended to be used for storing data, the std::fstream::out flag would be used as such:

#include <fstream>

std::fstream important_data("/path/to/my/file", std::fstream::out)

However, this method will always truncate (empty) the file. Another flag, std::fstream::app, will allow us to set the file to be appended to when written to. This is not placed as a third argument to the std::fstream constructor, but it is binary OR’d (|) with all flags which are to be set. Therefore, if the user wishes to open the file for writing and to append each written string to the file, the following flags would be set in the constructor:

#include <fstream>

std::fstream important_data("/path/to/my/file", std::fstream::out | std::fstream::app)

Writing strings to the file stream

#include <fstream>

std::fstream important_data("/path/to/my/file")
important_data << 1 << ", " << 'z' << std::endl;

std::fstream::close

#include <fstream>

std::fstream important_data("/path/to/my/file")
important_data << 1 << ", " << 'z' << std::endl;
important_data.close();

Closing the file stream ensures that the file is correctly and safely closed with respect to the underlying operating system. All files which are opened must be closed.

fstream Functionality Example — Implementation

// DNSBackup.h
#ifndef DNS_BACKUP_H
#define DNS_BACKUP_H

#include <fstream>
#include <sstream>

namespace DNS {
  class DNSEntry {
    public:
      // initialise the entry with blank fields
      DNSEntry() : DNSEntry(std::string(""), std::string("")) {}
      // initialise the entry with pre-determined fields
      DNSEntry(std::string domain, std::string ip) : domain(domain), ip(ip) {}

      // output stream overload
      friend std::ostream& operator<<(std::ostream& out, const DNSEntry& entry) {
        out << entry.domain << "\t" << entry.ip << std::endl;
        return out;
      }

      // input stream overload
      friend std::istream& operator>>(std::istream& in, DNSEntry& entry) {
        std::string working;
        // while there are still unread strings which can be read in
        if (std::getline(in, working)) {
          // convert to a string stream (what is a string stream?)
          std::stringstream ss(working);
          ss >> entry.domain;
          ss >> entry.ip;
        }
        return in;
      }

      std::string get_domain() { return this->domain; }
      std::string get_ip() { return this->ip; }
    private:
      std::string domain, ip;
  };

  class DNSBackup {
    public:
      // constructor with a default file argument
      DNSBackup() : DNSBackup(std::string("dns_backup.txt")) {}
      DNSBackup(std::string file) {
        // acquire resource (file)
        this->backup_fs.open(file, std::fstream::in | std::fstream::out | std::fstream::app);
      }

      // MEMBER output operator method overload - what is different about this and the regular
      // std::ostream operator overload?
      bool operator<<(const DNSEntry& entry) {
        return this->backup_fs << entry ? true : false;
      }

      // MEMBER input operator method overload - what is different about this and the regular
      // std::istream operator overload?
      bool operator>>(DNSEntry& entry) {
        return this->backup_fs >> entry ? true : false;
      }

      ~DNSBackup() {
        // release resource (file), which is invoked when this object leaves scope (or is deleted)
        this->backup_fs.close();
      }

    private:
      std::fstream backup_fs;
  };
}

#endif // DNS_BACKUP_H

// main.cpp
int main (void) {

  std::string backup_file("dns_example_backup.txt");
  DNS::DNSBackup *backup_manager = new DNS::DNSBackup(backup_file);

  std::vector<DNS::DNSEntry> entries;
  entries.push_back(DNS::DNSEntry("dns.somesite.com", "123.123.123.1"));
  entries.push_back(DNS::DNSEntry("somesite.com", "123.123.123.2"));
  entries.push_back(DNS::DNSEntry("another.site", "122.3.123.1"));
  entries.push_back(DNS::DNSEntry("example.com.au", "123.128.123.1"));
  std::vector<DNS::DNSEntry>::iterator entry;
  for (entry = entries.begin(); entry != entries.end(); ++entry) {
    bool success = *backup_manager << *entry;
    if (success) {
      std::cout << "Adding: ";
    } else {
      std::cout << "Failed to add: ";
    }
    std::cout << *entry;
  }

  // force the object to leave scope by deleting it
  delete backup_manager;

  DNS::DNSBackup *backup_manager_revived = new DNS::DNSBackup(backup_file);

  DNS::DNSEntry dns_entry;
  while (*backup_manager_revived >> dns_entry) {
    std::cout << "Recieved: " << dns_entry.get_domain();
    std::cout << ", " << dns_entry.get_ip() << std::endl;
  }

  return EXIT_SUCCESS;
}

Templates

Templates are a construct available in C++ which allows type-strict C++ code to handle generic types. This means that the type which the class can handle is not known until compile time. At compile time, the classes / types which declared as an input type (using the <...> notation) to the templated class then receive a generated copy of the templated code which substitutes the generic types (in the case below, all mentions of T) with the input type.

// ExampleTuple.h
#include <iostream>

namespace Week8 {
  template <typename T>
  class Tuple {
    public:
      Tuple(T first, T second) : first(first), second(second) {}

      T get_first() { return this->first; }
      T get_second() { return this->second; }
    private:
      T first, second;
  };
}
Week8::Tuple<int> simple_tuple(2, 3);

The above code snippet creates an object of the Tuple type using the int type. All occurrences of T are then replaced with int.

Week8::Tuple<char> char_tuple('a', 'z');

Likewise, the above code snippet creates an object of the Tuple type using the char type. All occurrences of T are then replaced with char. These two constructors can be called in the same file as int and char versions are created separately from each other.


namespace Week8 {
  template <typename T, typename U>
  class BetterTuple {
    public:
      BetterTuple(T first, U second) : first(first), second(second) {}
      T get_first() { return this->first; }
      U get_second() { return this->second; }
    private:
      T first;
      U second;
  };
}

Since two typenames are used in the template declaration, all uses of T are substituted with the first input type and all uses of U are substituted with the second input type.

Week8::BetterTuple<int, char> nice_tuple(8, 'a');

The above code snippet creates an object of the BetterTuple type using the int and char types. All occurrences of T are then replaced with int, and all occurrences of U are then replaced with char.

Week8::BetterTuple<int, DNS::DNSEntry> custom_tuple('a', entries[0]);

The above code snippet creates an object of the BetterTuple type using the int and DNS::DNSEntry (which is a custom type) types. All occurrences of T are then replaced with int, and all occurrences of U are then replaced with DNS::DNSEntry.
